SCPDiscoveryConfiguration


@interface SCPDiscoveryConfiguration : NSObject

Configuration for discovering readers.

  • -initWithDeviceType:

    Initializes a discovery configuration with the given device type. The the discovery method is the default for that device type.

    Declaration

    Objective-C

    - (nonnull instancetype)initWithDeviceType:(SCPDeviceType)deviceType;

    Swift

    init(deviceType: DeviceType)

    Parameters

    deviceType

    The device type to discover.
  • -initWithDeviceType:method:

    Initializes a discovery configuration with the given device type and discovery method.

    Note that this initializer returns an optional value because not all deviceType and method combinations are valid. For the Chipper 2X, we recommend using the bluetoothProximity discovery method, which lets you discover a single reader by holding it next to the iOS device.

    Declaration

    Objective-C

    - (nullable instancetype)initWithDeviceType:(SCPDeviceType)deviceType
                                     method:(SCPDiscoveryMethod)method;

    Swift

    init?(deviceType: DeviceType, method: DiscoveryMethod)

    Parameters

    deviceType

    The device type to discover.
    method

    The discovery method to use.
  • timeout

    The timeout (in seconds) after which discoverReaders should fail. If the value is 0 (the default), discoverReaders will never timeout.

    Note that setting a timeout is not currently supported when using:

    • The BluetoothProximity DiscoveryMethod
    • The ReaderSimulator DeviceType
